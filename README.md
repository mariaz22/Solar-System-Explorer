# Solar System Explorer

A real-time solar system simulation built in Unity. The project covers three technical areas in depth: a physics simulation grounded in real gravitational mechanics with custom rocket dynamics and particle systems, an AI system built around a seven-state finite state machine and A\* pathfinding that drives a fully autonomous probe through the system, and a custom rendering pipeline of hand-written HLSL shaders covering procedural cloud generation, atmospheric effects, and screen-space distortion. On top of that, the whole solar system can be fast-forwarded through 13 billion years of stellar evolution while the sun's physics, appearance, and gravitational influence update in real time.

## Getting started

Open `Assets/Scenes/SampleScene.unity` in Unity (URP, Unity 6 or later) and hit Play. The bootstrap system handles everything: it lays out the solar system, applies materials, starts orbital motion, configures lighting, and builds the HUD. There are no manual configuration steps.

If the planet materials are missing or need to be regenerated, run `Tools > Solar System > Create Planet Materials` from the Unity menu bar. This bakes the material assets into `Assets/Resources/Materials/Planets/`.

## Physics simulation

Orbital motion uses Kepler's third law to derive angular speeds — inner planets orbit faster, outer ones slower, and the ratios are physically correct. That gives the simulation its backdrop, but a separate N-body gravity system runs on top of it. `GravitySimulator` applies Newton's law of gravitation every physics step to every object tagged as `CelestialBody`, so the probe and dynamic bodies experience real gravitational pull from every planet simultaneously. In manual control mode you feel this directly: the probe drifts toward nearby massive bodies and you have to compensate with thrust.

The probe itself has a simulated propulsion system built from scratch. `ProceduralRocket` generates the rocket geometry at runtime and `RocketExhaust` drives a multi-layered particle exhaust — an inner flame on a separate child object, an outer plume, and heat shimmer. Thrust, boost, and braking each produce different exhaust states. Solar wind (`SolarWind`) applies a continuous directional force to all celestial bodies, and rogue asteroids spawned by the event system fly across the system on real physics trajectories rather than scripted paths. Meteor showers spawn clusters of physics projectiles that scatter on impact. Planetary destruction during the Red Giant phase uses an explosion particle system (`PlanetExplosionVFX`) with forces that push debris outward from the blast center.

## Probe intelligence

The probe's behavior is driven by a seven-state finite state machine. `Idle` waits for a mission. `ChooseTarget` scores all unvisited planets against an exploration priority function and picks the best one. `Travel` moves the probe along a path generated by the A\* pathfinder, applying smooth-damping for natural deceleration and adjusting speed dynamically when passing through nebula regions. `Scan` holds the probe in orbit for two seconds, triggers a scanning ring animation, and writes the planet data to the mission log. `AvoidCollision` activates when a forward raycast hits an obstacle — it steers clear for three seconds and then hands back to Travel. `Return` brings the probe home once all planets are explored. `ManualControl` switches to direct piloting while keeping gravity and solar wind active.

`AStarPathfinder` generates waypoints by treating each planet as a navigational obstacle with a mass-weighted traversal cost — heavier planets cost more to route through, so the probe naturally avoids flying deep into Jupiter's or Saturn's gravity wells when there's a longer but safer path. The pathfinder builds a graph of waypoint shells around each planet and finds the minimum-cost route between them. The reactive nebula system (`ReactiveNebula`) gives each of the four outer nebula volumes three internal states — Calm, Warning, and Danger — that change based on probe proximity. The probe reads the current state to decide how aggressively to slow down, and `NebulaVisionEffect` tints the screen to match.

## Custom rendering

The rendering pipeline is built around a set of custom shaders written in HLSL for URP rather than using Unity's built-in material system.

Planetary atmospheres use `FresnelAtmosphere`, a rim-light shader that scales the glow edge to each planet's atmospheric density. Airless bodies like Mercury get none. Planets with atmospheres also get a cloud layer rendered by `CloudLayer`, which generates cloud shapes using four-octave 3D Fractional Brownian Motion noise sampled from the world-space surface normal. Using the normal instead of UV coordinates eliminates the seam artifact that 2D noise produces at the wrap edge of sphere meshes. The clouds scroll over time, fade with a quadratic falloff at the silhouette edge to avoid a halo ring, and go completely dark on the night side — the sun direction is computed per-fragment from world position rather than using the engine's light API, which only returns directional lights and would ignore the point-light sun. Each planet has its own cloud color, density, scroll rate, and noise scale tuned individually.

The Sun is surrounded by a screen-space heat distortion zone rendered by `SunHeatDistortion`. The shader reads the opaque scene texture captured before the transparent render pass and displaces the sample coordinates using two superimposed sinusoidal layers — a slow coarse roll and a faster shimmer. The sphere is rendered with front-face culling and a depth test that fails where the sun body occupies the depth buffer, so the distortion appears only in the space around the corona and never overlaps the sun itself. The distortion intensity and warm tint scale with the current solar evolution stage. The two billboard quad layers around the sun (glow and corona) use additive blending so they feed directly into the bloom pass.

Post-processing is assembled at runtime: ACES tonemapping, a vignette, a bloom pass calibrated to catch the sun and any emissive surfaces, and FXAA. Two very dim directional fill lights prevent the fully-black night sides that a single point light would otherwise produce without contributing visible directionality of their own.

Procedural generation handles everything that doesn't have a real texture: `ProceduralPlanetTexture` generates albedo and normal maps from layered noise for planets that lack source images, Saturn's rings are a procedurally built transparent mesh, and `Starfield` places thousands of stars across the skybox at runtime.

## Cosmic timeline

`CosmicTimelineManager` maps the simulation clock to five solar stages across 13 billion years: Main Sequence, Sub-Giant, Red Giant, Planetary Nebula, and White Dwarf. `SunEvolutionController` drives the sun's scale, light intensity, color temperature, and emission per stage, and updates the heat distortion intensity through `SunDistortionSetup`. `PlanetEvolutionController` adjusts each planet's atmosphere and scale, and detonates planets that wouldn't survive the Red Giant expansion. The UI timeline slider scrubs through the full lifecycle in real time.

## Random events

`RandomEventManager` fires three encounter types at randomized intervals regardless of where the probe is or what stage the timeline is at. Solar storms push nearby objects outward with a burst of force. Rogue asteroids cross the system on real ballistic trajectories. Meteor showers spawn clusters of physics projectiles aimed at the inner system. All three show up in the mission log with timestamps.

## Controls

During Play Mode the probe operates autonomously. Manual control is available from the Planet Selection panel or by taking over input directly.

**Camera** — right-click and drag to orbit, scroll to zoom, WASD to pan. The camera auto-focuses on the selected planet when the probe arrives.

**Manual probe control** — WASD for horizontal movement, Q/E for vertical, left shift to boost. Space triggers a manual scan near a planet.

**Time scale** — the bottom-right slider runs from 1× to 100×. Pause freezes everything except the UI.

**Cosmic timeline** — the upper slider scrubs through the 13 Gyr lifecycle. "Present Day" snaps back to 4.5 Gyr.

**Planet selection** — the top-left dropdown lists all planets. Selecting one shows orbital data and surface facts, and "Send Probe" dispatches the probe immediately.

```
  Resources/
    Materials/Planets/  Pre-baked planet .mat files

  Scenes/               SampleScene.unity — the only scene
  Scripts/
    Camera/             CameraController, FreeFlyCamera
    Events/             RandomEventManager, MeteorMover
    Pathfinding/        AStarPathfinder, PathVisualizer
    Planets/            Planet, PlanetData, PlanetManager
    Probe/              ProbeController, ProceduralRocket, RocketExhaust
      States/           IdleState, ChooseTargetState, TravelState, ScanState,
                        AvoidCollisionState, ReturnState, ManualControlState
    Simulation/         OrbitalMotion, GravitySimulator, CelestialBody,
                        SunEvolutionController, PlanetEvolutionController,
                        SunDistortionSetup, CosmicTimelineManager,
                        ReactiveNebula, AsteroidBelt, SaturnRings,
                        Starfield, SolarWind, and others
    UI/                 HUDController, PlanetSelectionUI, StartScreen,
                        TimeScaleController, MissionLog, CosmicTimelineUIController
  Shaders/              FresnelAtmosphere.shader, CloudLayer.shader,
                        SunHeatDistortion.shader, PlanetSurface.shader
  Settings/             URP renderer and volume profiles
```
